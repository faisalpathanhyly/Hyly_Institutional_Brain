# GitHub Editing Guide

## Purpose

This guide explains how non-developers can make small changes directly in GitHub.

## Small Text Changes

To update a rule, prompt, chapter, or knowledge asset:

1. Open the repository in GitHub.
2. Navigate to the file.
3. Click the edit button.
4. Make the change.
5. Add a clear commit message.
6. Commit the change.
7. Add the update to `history.md` if it is meaningful.

## ZIP Files

ZIP files are only needed for initial import or large rebuilds.

Do not re-import a ZIP for every small change.

## Example Commit Message

```text
Update evaluation rule for introduction_to_multifamily
```
