# Review Workflow

## Purpose

The review workflow helps keep the Institutional Brain trusted and usable.

## Basic Workflow

```text
Draft
↓
Internal Review
↓
Approved
↓
Published
↓
Maintained
```

## When Review Is Needed

Review is needed when:

- A new knowledge domain is added
- A major folder structure changes
- A course rule changes
- A prompt changes
- A knowledge asset is intended for learner or client use
- A source of truth changes

Small typo fixes do not require formal review but should still use a clear commit message.
