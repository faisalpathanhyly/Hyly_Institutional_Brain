# Chapter 2: Delegation — Should AI Do This?

This chapter introduces the first D: **Delegation**. Delegation is the decision you make before opening any AI tool.

The goal is to avoid two common problems:

- Giving AI work it should not own
- Doing manually what AI could handle faster

## 2.1 What Delegation Means

Delegation is the first D because AI fluency starts before prompting. A fluent AI user does not open a tool first and think later.

They first ask:

- Should AI do this?
- Which part should AI do?
- Which part should stay with me?
- Which tool or model fits the work?

**Delegation = Deciding WHAT to give AI, WHICH tool to use, and WHAT to keep human — before you open any AI tool.**

### Knowledge Check

**Question:** A teammate says, “I always open AI first and figure out the task while typing.” What is the best Delegation advice?

A. Open AI first because the tool should decide what the task is.

B. Pause first and decide what AI should do, what you should do, and which tool fits the task.

C. Avoid using AI because thinking before prompting takes too much time.

D. Use the strongest model every time so Delegation is unnecessary.

**Correct Answer:** B — Pause first and decide what AI should do, what you should do, and which tool fits the task, is correct because Delegation happens before prompting.

## 2.3 Execution vs. Judgment

| Task Type | Give to AI? | Why? |
|---|---|---|
| Repetitive, pattern-based work | Yes | AI is fast, consistent, and tireless. |
| Data-heavy formatting or mapping | Yes | Clear rules, no nuance needed. |
| Drafting first versions of content | Yes | AI generates fast, and the learner refines. |
| Decisions with business context | No | Requires human knowledge of relationships and stakes. |
| Strategic or sensitive conversations | No | Trust and tone matter. AI cannot read the room. |
| Defining standards or policies | No | Judgment work comes from experience, not rules. |

### Knowledge Check

A. Yes, because this is structured execution work with clear rules.

B. No, because AI should only be used for sensitive business decisions.

## 2.4 Delegation in Hyly Workflows

| Task | Delegate? | Why? |
|---|---|---|
| Creating E2E test scripts from requirements | Yes | Pure execution — Claude does it in hours vs. days. |
| Mapping bugs from QA Tracker to PM.Bugs | Yes | Structured data with clear rules — no judgment needed. |
| Detecting flaky tests across hundreds of runs | Yes | Pattern recognition — AI excels here. |
| Defining TDD standards for Hyly | No | Judgment work — came from team experience, not AI. |
| Deciding if a P0 bug is really P0 | No | Business impact assessment — only a human PM knows. |
| Noticing when a team skips quality steps | No | Team dynamics and culture — AI sees numbers, not people. |

### Model Selection Is Also Delegation

| Model | Best for | Skip when |
|---|---|---|
| **Haiku** | High-volume, fast tasks — formatting, classification, simple Q&A | Complex reasoning or nuanced judgment is needed |
| **Sonnet** | Everyday work — drafting, analysis, research, most workflows | Highest-quality reasoning is critical |
| **Opus** | Deep analysis, complex multi-step reasoning, critical output | Speed or cost is a constraint |

### Knowledge Check

**Question:** In a QA workflow, Claude maps bugs from QA Tracker into PM.Bugs using clear field rules. But a PM still decides whether a bug is truly P0. Why is this the right Delegation split?

A. AI can handle structured mapping, but business severity needs human judgment.
